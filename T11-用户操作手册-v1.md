## 1. 系统简介

本系统是一个基于AI规则引擎的内部存货交易未实现利润自动抵消工具。用户只需准备三份Excel表格，上传后系统自动完成：

- 识别每笔内部交易的类型（顺流/逆流/平流）
- 计算未实现内部销售利润
- 生成标准合并抵消分录（含递延所得税 + 少数股东分摊）
- 导出抵消分录汇总报告

**适用场景**：集团合并报表编制时，替代手工Excel计算内部存货交易的抵消分录。

## 2. 环境准备与安装

### 2.1 前置条件

- Python 3.11 或更高版本
- Windows / macOS / Linux 均可

### 2.2 安装步骤

**第一步：下载项目文件**

将老师提供的项目压缩包解压到本地任意目录。

**第二步：安装依赖**

打开命令行（终端），进入项目目录，执行：

```bash
pip install -r requirements.txt
```

系统将自动安装以下依赖：
- `streamlit` — Web界面框架
- `pandas` — 数据处理
- `openpyxl` — Excel读写
- `reportlab` — PDF生成

安装过程约1-2分钟。

### 2.3 验证安装

```bash
streamlit --version
```

如果显示版本号，安装成功。

## 3. 启动系统

在项目目录下执行：

```bash
streamlit run app.py
```

系统自动打开浏览器，显示主界面。

> 若浏览器未自动打开，手动访问终端提示的地址（通常为 `http://localhost:8501`）。

## 4. 操作流程（分步详解）

### 步骤1：上传股权关系配置表

**目的**：告诉系统集团内各公司之间的母子关系。

点击「上传股权关系表」按钮，选择Excel文件。

**文件格式**：

| 母公司编码 | 子公司编码 | 母公司持股比例 | 是否纳入合并 |
|-----------|-----------|:-----------:|:----------:|
| P001 | S001 | 0.80 | 是 |
| P001 | S002 | 0.60 | 是 |

**操作要点**：
- 这是**一次性配置**，首次使用后系统会记住，无需每月重复上传
- 持股比例填写小数（如80%填0.80）
- "是否纳入合并"填写"是"或"否"

上传成功后，系统显示股权关系总览表。

---

### 步骤2：上传内部销售明细表

**目的**：提供本月合并范围内各主体之间的存货销售数据。

点击「上传内部销售明细表」按钮，选择Excel文件。

**文件格式**：

| 销售方编码 | 购买方编码 | 存货批次号 | 存货名称 | 销售收入 | 销售成本 | 销售数量 | 会计期间 |
|-----------|-----------|-----------|---------|:------:|:------:|:------:|---------|
| P001 | S001 | B202605001 | A产品 | 1000000 | 600000 | 100 | 202605 |
| S001 | S002 | B202605002 | B材料 | 500000 | 350000 | 200 | 202605 |

**操作要点**：
- 金额单位为元（不含税）
- "存货批次号"需要与步骤3的结存表一一对应
- 多个月份的数据可分次上传（勾选"追加"模式），或一次性覆盖（默认）
- 上传后系统自动校验数据完整性，如有缺失列会提示

---

### 步骤3：上传存货期末结存表

**目的**：告诉系统每批内部购入的存货还有多少没卖出去。

点击「上传存货结存表」按钮，选择Excel文件。

**文件格式**：

| 主体编码 | 存货批次号 | 期末结存数量 | 内部购入总数量 |
|---------|-----------|:--------:|:----------:|
| S001 | B202605001 | 70 | 100 |
| S002 | B202605002 | 50 | 200 |

**操作要点**：
- "主体编码"填写存货持有方（即购买方）
- "期末结存数量"指截至月末尚未对外售出的数量
- "内部购入总数量"指该批次最初从内部购买的总数量
- 如果某个批次已全部对外售出（结存=0），可以不填该行；系统会将对应交易标记为"已全部实现，无需抵消"

---

### 步骤4：查看数据校验结果

三份表格上传完成后，系统自动进行交叉校验，并展示校验报告：

**示例输出**：

```
═══════════════════════════════════
         数据校验报告
═══════════════════════════════════
✅ 内部销售明细：15 笔交易
✅ 存货结存记录：12 条
✅ 股权关系：3 组母子关系

⚠ 警告：
  - 批次 B202605008（销售方S003→购买方S004）
    交易对手未在股权关系表中找到 → 标记为"未识别"
  - 批次 B202605010 有销售记录但无结存记录
    → 视为已全部对外售出

❌ 错误：无
═══════════════════════════════════
```

如发现错误（如必填列缺失），请修正Excel文件后重新上传。

---

### 步骤5：执行抵消计算

点击「🚀 执行抵消计算」按钮。

系统自动完成以下步骤（进度条实时显示）：

1. 🟢 交易类型识别中...
2. 🟢 未实现利润计算中...
3. 🟢 抵消分录生成中...

计算耗时：100笔交易以内约2-5秒。

## 5. 常见问题

### Q1：上传Excel后提示"列名不匹配"？

请对照附录A的模板格式检查列名。列名必须**完全一致**（包括空格和标点）。

### Q2：计算结果中某笔交易标记为"未识别"？

说明该交易的销售方和购买方在股权关系表中找不到母子关联。请检查：
- 双方编码是否在股权关系表中存在
- 间接持股关系是否配置（如A→B，B→C，则A和C的交易需配置A→C的间接关系）

### Q3：想修改某笔抵消分录的金额？

在分录明细行右侧点击「✏️调整」，输入修正后的未实现利润金额，点「重算」。原计算结果和修改后结果都会保留在日志中。

### Q4：税率可以修改吗？

可以。在左侧边栏「⚙️ 系统配置」中修改所得税税率（默认25%）。修改后需重新点击"执行抵消计算"。

### Q5：如何清空历史数据重新开始？

删除项目目录下 `data/` 文件夹，重启系统即可。
