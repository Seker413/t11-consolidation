## 1. 引言

### 1.1 背景与编制目的

在集团型企业中，母子公司之间、子公司相互之间经常发生存货购销交易。按照《企业会计准则第33号——合并财务报表》的规定，编制合并报表时必须将内部交易产生的营业收入、营业成本予以抵消，同时对期末存货中包含的未实现内部销售利润予以抵消，并确认递延所得税影响。

传统手工抵消过程依赖财务人员在Excel中逐笔勾对交易、手工计算未实现利润和编制抵消分录，存在以下痛点：

- **工作量大**：集团内部交易数据分散在多家法人主体的账簿中，逐笔匹配耗时漫长
- **易出错**：顺流/逆流/平流交易的抵消比例不同、连续编报的期初滚调复杂，人工计算容易遗漏
- **追溯困难**：审计、巡视需要追溯每笔抵消的计算过程，手工痕迹难以完整还原
- **与穿透监管要求脱节**：国资委1号文、2号文要求实现"数据同源、凭证一穿到底"，手工抵消无法满足智能化监管闭环

本系统旨在运用AI规则引擎技术，实现内部存货交易未实现利润的自动识别、自动计算、自动生成抵消分录。

### 1.2 适用范围（V1.0）

| 维度 | V1.0范围 |
|------|----------|
| 交易类型 | 顺流交易（母→子）、逆流交易（子→母）、平流交易（子↔子） |
| 存货计价 | 个别计价法、先进先出法（FIFO） |
| 税务处理 | 递延所得税自动计算 |
| 少数股东 | 逆流/平流场景自动分摊 |
| 输入方式 | Excel模板上传（内部销售表 + 存货结存表） |
| 输出方式 | 抵消分录在线展示 + Excel/PDF导出 |

**V1.0暂不包含**：加权平均法支持、存货跌价准备抵消、连续编报期初滚调、XBRL输出、多系统API对接。

### 1.3 术语与定义

| 术语 | 定义 |
|------|------|
| **内部存货交易** | 合并范围内法人主体之间发生的存货购销业务 |
| **未实现内部销售利润** | 内部销售方已确认利润，但从集团整体角度看存货尚未对外售出，该利润未实现 |
| **顺流交易** | 母公司向子公司销售，未实现利润全部归属于母公司股东 |
| **逆流交易** | 子公司向母公司销售，未实现利润在母公司与少数股东之间按持股比例分摊 |
| **平流交易** | 子公司之间相互销售，未实现利润在销售方母公司与少数股东之间分摊 |
| **抵消分录** | 在合并底稿层面编制的调整分录，不影响各法人主体的独立账簿 |
| **AI规则引擎** | 基于可配置业务规则（非硬编码）自动判断交易类型、选择抵消模板、执行计算逻辑的核心组件 |

### 3.2 AI规则引擎——交易类型自动识别（SRS-F02）

#### 3.2.1 识别逻辑

系统读取股权关系配置表，构建母子公司关系图谱，对每笔内部交易自动判定：

```
输入：销售方A、购买方B

规则1：若 A 是 B 的母公司 → "顺流交易"
规则2：若 B 是 A 的母公司 → "逆流交易"
规则3：若 A 和 B 有共同母公司 → "平流交易"
规则4：无法判定 → 标记"未识别"，提示用户手工确认
```

#### 3.2.2 少数股东比例提取

根据股权关系表自动获取：

```
逆流交易：少数股东比例 = 1 - 母公司对销售方（子公司）的持股比例
平流交易：少数股东比例 = 1 - 销售方母公司的持股比例
```

### 3.3 未实现利润自动计算引擎（SRS-F03）

#### 3.3.1 核心公式

```
毛利率 = (销售收入 - 销售成本) / 销售收入
未对外出售比例 = 期末结存数量 / 内部购入总数量
未实现利润 = 销售收入 × 毛利率 × 未对外出售比例
```

#### 3.3.2 三种计算场景

**场景一：期末全部未对外出售（未出售比例 = 100%）**

```
未实现利润 = 销售收入 - 销售成本
```

**场景二：期末部分售出**

```
已实现部分 = 销售收入 × 毛利率 × 已出售比例 → 不抵消
未实现部分 = 销售收入 × 毛利率 × 未出售比例 → 需抵消
```

**场景三：期末已全部对外售出（结存数量 = 0）**

```
未实现利润 = 0 → 该笔交易无需抵消，跳过
```

#### 3.3.3 成本流转假设支持

| 方法 | V1.0实现方式 |
|------|-------------|
| 个别计价法 | 按批次号精确匹配，直接使用销售明细表中该批次的实际成本 |
| 先进先出法（FIFO） | 当同一存货有多个批次时，按购入顺序判断期末库存来自哪些批次 |

### 3.4 抵消分录自动生成（SRS-F04）★核心展示功能

#### 3.4.1 分录生成模板

**模板一：顺流交易（母→子）——未实现利润全部归母**

```
借：营业收入          [销售收入 × 未出售比例]
  贷：营业成本        [倒挤差额]
  贷：存货            [未实现利润]

借：递延所得税资产    [未实现利润 × 25%]
  贷：所得税费用      [同上]
```

**模板二：逆流交易（子→母）——涉及少数股东**

```
借：营业收入          [销售收入 × 未出售比例]
  贷：营业成本        [倒挤差额]
  贷：存货            [未实现利润]

借：递延所得税资产    [未实现利润 × 25%]
  贷：所得税费用      [同上]

借：少数股东权益      [未实现利润 × (1-25%) × 少数股东持股%]
  贷：少数股东损益    [同上]
```

**模板三：平流交易（子↔子）——与逆流同理**

```
借：营业收入          [销售收入 × 未出售比例]
  贷：营业成本        [倒挤差额]
  贷：存货            [未实现利润]

借：递延所得税资产    [未实现利润 × 25%]
  贷：所得税费用      [同上]

借：少数股东权益      [未实现利润 × (1-25%) × 销售方少数股东持股%]
  贷：少数股东损益    [同上]
```

### 3.7 异常数据自动检测（SRS-F07）✅ 已实现

#### 3.7.1 描述

系统在数据上传后、抵消计算前，自动扫描数据中的异常情况，并生成预警报告。

#### 3.7.2 检测规则

| 异常类型 | 检测逻辑 | 系统行为 |
|----------|----------|----------|
| 库存数据缺失 | 内部销售明细中的批次在存货结存表中无对应记录 | ⚠ 警告：可能已全部售出或数据缺失 |
| 毛利率异常 | 销售成本 > 销售收入（负毛利） | ⚠ 警告：可能为亏损销售或数据录入错误 |
| 股权关系缺失 | 交易双方均不在股权关系表中 | ⚠ 警告：无法判定交易类型 |
| 结存数据异常 | 期末结存数量 > 内部购入总数量 | ⚠ 警告：数据逻辑矛盾 |

#### 3.7.3 异常处理策略

- 异常不阻断计算流程
- 异常交易仍参与抵消（按保守策略处理：未知类型按顺流处理、缺失库存视为已全部售出）
- 所有异常汇总展示在仪表盘顶部，并写入导出报告

## 4. V2.0技术方案

### 4.1 技术栈

| 层次 | V1.0选型 | 说明 |
|------|----------|------|
| 编程语言 | Python 3.11+ | 财务计算生态成熟（pandas/openpyxl） |
| Web框架 | Streamlit | 纯Python即可构建交互界面，无需前后端分离 |
| 数据存储 | SQLite | 零配置本地数据库，计算日志持久化 |
| 计算引擎 | Python + JSON规则配置 | 抵消规则以JSON配置文件存储，体现"规则引擎"思想 |
| Excel处理 | openpyxl / pandas | 读写Excel模板 |
| PDF生成 | ReportLab | 抵消分录报告导出 |
| 版本控制 | Git | 代码版本管理 |

### 4.2 为什么选Streamlit而不是Java+Vue？

| 对比维度 | Streamlit（V1.0） | Java Spring Boot + Vue（V2.0+） |
|----------|-------------------|-------------------------------|
| 开发效率 | 极高，1-2天可出原型 | 需搭建前后端框架，至少1-2周 |
| 学习成本 | Python基础即可 | 需Java/Spring/Vue/数据库多项技能 |
| 交付物 | 一个Python脚本+数据文件 | 多个子项目+部署脚本+数据库脚本 |
| 适合场景 | **原型验证、算法展示、课程作业** | 企业级生产系统 |
| 老师关注点 | 业务逻辑正确性、AI协作能力 | 系统架构、安全合规 |

**V1.0的目标是让抵消分录"跑起来"，让老师看到核心算法是正确且完整的。** 企业级架构留作文档中的"未来扩展"描述。

### 4.3 系统架构（V2.0 模块化）

```
项目结构：
├── app.py                         # Streamlit主界面（仪表盘+交互）
├── rules.json                     # AI规则引擎配置
│
├── engine/                        # 核心计算引擎
│   ├── trade_classifier.py       # 交易类型分类 + 异常检测
│   ├── calc_engine.py            # 未实现利润计算 + 汇总统计
│   └── entry_generator.py        # 抵消分录生成（JSON规则驱动）
│
├── utils/                         # 工具模块
│   ├── hash_utils.py             # SHA256哈希链（可追溯）
│   └── pdf_export.py             # PDF报告导出（中文支持）
│
├── templates/                     # Excel模板（含示例数据）
├── data/                          # SQLite数据库（日志持久化）
└── .streamlit/                    # Streamlit配置

数据流：
  Excel上传 → 数据校验 → 异常检测 → 类型分类 → 利润计算
     → 分录生成 → 哈希存证 → 仪表盘展示 → Excel/PDF导出
```

**模块化设计优势**：各模块职责单一、可独立测试、可替换。新增交易类型只需修改 `rules.json`，无需改动核心代码。

## 6. 数据模型（V1.0）

### 6.1 SQLite表设计

**交易明细表（trade_detail）**

```sql
CREATE TABLE trade_detail (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    batch_id TEXT NOT NULL,          -- 批次ID（每次上传生成）
    seller_code TEXT NOT NULL,       -- 销售方编码
    buyer_code TEXT NOT NULL,        -- 购买方编码
    batch_no TEXT NOT NULL,          -- 存货批次号
    item_name TEXT,                  -- 存货名称
    revenue DECIMAL(18,2),           -- 销售收入
    cost DECIMAL(18,2),              -- 销售成本
    quantity INTEGER,                -- 销售数量
    period CHAR(6),                  -- 会计期间
    trade_type TEXT,                 -- 交易类型（DOWN/UP/FLAT/UNKNOWN）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**抵消分录表（elim_entry）**

```sql
CREATE TABLE elim_entry (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    elim_no TEXT NOT NULL,           -- 分录编号
    batch_id TEXT NOT NULL,
    trade_detail_id INTEGER,         -- 关联交易明细
    dr_cr CHAR(1),                   -- D/C
    account_name TEXT,               -- 科目名称
    amount DECIMAL(18,2),            -- 金额
    calc_params TEXT,                -- 计算参数JSON
    log_hash TEXT,                   -- 本地哈希
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 9. 附录

### 附录A：完整计算示例

**场景：母公司P（持股子公司S 80%）向S销售A产品100件，售价100万元（1万/件），成本60万元（0.6万/件）。S期末将其中30件对外售出，70件仍为库存。所得税税率25%。**

**输入（Excel数据）：**

表A-内部销售明细：
| 销售方 | 购买方 | 批次号 | 品名 | 销售收入 | 销售成本 | 数量 | 期间 |
|--------|--------|--------|------|----------|----------|------|------|
| P001 | S001 | B001 | A产品 | 1,000,000 | 600,000 | 100 | 202605 |

表B-存货结存：
| 主体 | 批次号 | 期末数量 | 购入总量 |
|------|--------|----------|----------|
| S001 | B001 | 70 | 100 |

表C-股权关系：
| 母公司 | 子公司 | 持股% | 合并 |
|--------|--------|-------|:---:|
| P001 | S001 | 80% | 是 |

**计算过程：**

```
步骤1：交易类型识别
  P001是S001的母公司 → 顺流交易

步骤2：未实现利润计算
  毛利率 = (1,000,000 - 600,000) / 1,000,000 = 40%
  未出售比例 = 70 / 100 = 70%
  未实现利润 = 1,000,000 × 40% × 70% = 280,000 元

步骤3：抵消分录生成（顺流交易模板）
  借：营业收入                        700,000  （100万 × 70%）
    贷：营业成本                      420,000  （倒挤）
    贷：存货                          280,000  （未实现利润）

  借：递延所得税资产                   70,000  （28万 × 25%）
    贷：所得税费用                     70,000

  顺流交易不涉及少数股东分摊。
```

```

  内部存货交易未实现利润抵消计算结果
  会计期间：2026年5月
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
